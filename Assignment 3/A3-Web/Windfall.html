Windfall

As a group, we decided to shift the “first level” from the extended features and integrated it as part of the Minimum Viable Features. We needed to optimise this because there was structural changes that occurred to the team at certain points in time that necessitated this shift. However, despite this adaptation, this optimised the efficiency of the completion of the assignment.

There were also some discrete workarounds the group discovered when trying to coordinate the assignment. For example, striking an appropriate balance between the designation of tasks and the duration it takes to complete them resulted in less time that it would take to complete and achieve certain milestones within the assignment. Also, using Github desktop was a streamlined and intuitive way to upload and edit files to Git. This avoided having to navigate to the website, login, and manually upload each file. Instead, the members of the team merely had to edit the files locally, and then forward it as a commit, which had a cumulative compounding effect in saving time.

Additionally, the construction of the timetable was a well-optimised effort because each group member focused on filling out the estimated time they would spend working on each applicable part of the project. Members of the group filled this out on a Google Drive document which all of the team had access to, which ensured that each member did not have to waste time creating a table or trying to format one. Therefore, this optimised the amount of time focused on productive work and not on work that does not contribute to the completion of the assignment within a timely manner.
